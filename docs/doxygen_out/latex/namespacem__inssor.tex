Sorts I\+N\+O\+U\+T\+V\+A\+L\+S() into ascending order (Insertion sort).

If certain requirements are met and performance is important this procedure can be far faster, but R\+E\+F\+S\+O\+R(3f) and ranking routines M\+R\+G\+R\+N\+K(3f) and M\+R\+G\+R\+E\+F(3f) are recommended for the general case.

This subroutine uses an Insertion sort. It does not use any work array and is faster when I\+N\+O\+U\+T\+V\+A\+L\+S() is of very small size ($<$ 20), or already almost sorted; but worst case behavior can be triggered by commonly encountered data order (e.\+g. initially inverse sorted).
